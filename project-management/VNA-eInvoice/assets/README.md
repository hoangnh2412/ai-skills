# Assets — Tài liệu dự án (VNA eInvoice)

Folder lưu **tài liệu thô** phục vụ phân tích — không phải artifact minipower (artifact → [`../docs/`](../docs/)).

## Cấu trúc

| Folder | Mô tả |
|--------|--------|
| [`public/`](public/) | Tài liệu, biên bản họp **với khách hàng VNA** (email, PDF, spec, MoM đã share) |
| [`internal/`](internal/) | Biên bản họp nội bộ, ghi chú team — **không gửi cho khách hàng** |

## Quy ước

| Quy tắc | Mô tả |
|---------|--------|
| **Phân loại đúng folder** | Có VNA tham gia / đã share → `public/` · chỉ nội bộ → `internal/` |
| **Giữ nguyên bản gốc** | Không sửa file gốc; phân tích → ghi vào `docs/` |
| **Đặt tên** | `YYYY-MM-DD_<mô-tả-ngắn>.<ext>` hoặc giữ tên file gốc nếu đã rõ ràng |
| **Trace** | Khi trích dẫn trong DOC, ghi nguồn: `assets/public/...` hoặc `assets/internal/...` |

## Artifact minipower

Tài liệu dự án (DOC-01–18) nằm tại [`../docs/`](../docs/) theo chuẩn [minipower](../../../.cursor/skills/minipower/SKILL.md).
